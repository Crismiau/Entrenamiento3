product API management system

important: make sure your JSON server is running on http://localhost:3000/products before using these functions.

main functions

getProducts()
retrieves all products from the API and displays them in a nice table format in the console. this function makes a GET request to fetch the current product list and shows it using console.table() which automatically formats the data into rows and columns for easy reading.
getProducts()
createProduct(productObject)
creates a new product by sending a POST request to the API. you need to provide a complete product object with all required fields: name, description, price (number), category, and stock (number). the function validates all fields before sending the request to prevent errors.
createProduct({
  id: 10,
  name: "gaming mouse",
  description: "high precision gaming mouse",
  price: 79.99,
  category: "accessories",
  stock: 15
})
updateProduct(id, updatedFields)
updates an existing product by its ID using a PUT request. you only need to include the fields you want to change - you don't have to provide all fields. the function validates that price and stock are numbers if included, and that text fields are not empty. the ID should be a string.
updateProduct("1", {
  price: 199.99,
  stock: 25
})
deleteProduct(id)
permanently removes a product from the API using a DELETE request. you just need to provide the product ID as a string. once deleted, the product cannot be recovered, so use this function carefully. the function will show a confirmation message when the deletion is successful.
deleteProduct("3")

test functions (easy to use)

testGet()
simple wrapper for getProducts(). just calls the get function to show all current products in the console table. use this when you want to quickly see what products exist in your database.
testGet()
testPost()
creates a predefined test product (PlayStation 6) without needing to type all the product details. this is useful for quickly testing the create functionality. the product has all required fields already filled in with sample data.
testPost()
testPut(id)
updates a product with predefined changes. it will change the product name to "PlayStation 6 Pro", update the description, set price to 900, and change stock to 30. you just need to provide the ID of the product you want to update.
testPut("2")
testDelete(id)
simple wrapper for deleteProduct(). just provide the ID of the product you want to remove. this makes it easier to test the delete functionality without typing the full function name.
testDelete("4")
createCustomProduct(name, description, price, category, stock)
convenient function to create products without writing the full object syntax. just provide the five parameters in order and it will create the product object automatically and send it to the API. makes creating products much faster and easier.
createCustomProduct("wireless headphones", "bluetooth headphones with noise canceling", 149.99, "audio", 20)
runAllTests()
executes a complete sequence of all CRUD operations automatically with delays between each step. it will: get all products, create a new product, update that product, delete it, and show final results. this is perfect for testing that all API endpoints are working correctly.
runAllTests()

how to find product IDs

to use update or delete functions, you need to know the product ID. follow these steps:

1. run testGet() or getProducts() first
2. look at the console table that appears
3. find the "id" column - these are the IDs you can use
4. use those IDs in your update/delete functions
5. remember IDs are strings, so use quotes: "1", "2", etc.

to execute this program, open the browser console (F12)
